Client policies progress: Draft → Staging → Production. Only production policies can be bound to subjects. Promote requires TOTP step-up.

Policy Dry-run / Simulate

Test a policy against a mock input scenario without applying any change. Returns block/allow verdict + deny codes + AI explanation.



Client-policy bindings (#16 — enforcement)

Bind a saved client policy (clients/<name>) to a subject scope and a direction. Bound policies are enforced by the gateway on every matching request (ingress) and/or response (egress), in addition to the core gates — they can only add denials, never remove one. Wildcard = leave Subject ID empty (all subjects of that kind).

Certificate Chain Detail —
CA Mode

Internal CA (default): certificates are issued by the built-in two-tier PKI (root → intermediate → per-service leaf). No external dependency.

BYO-CA mode: set YASHIGANI_PKI_CA_MODE=byo to use an external signing endpoint (step-ca, Vault PKI, etc.). Requires YASHIGANI_BYO_SIGNING_ENDPOINT and YASHIGANI_BYO_CA_CERT_PATH. See documentation for full configuration reference.
